📊 Data Flow Pipeline

StepConnectionProtocolDescription
1Patient → Daily SDKAudioBrowser captures voice via microphone, streams audio to Daily SDK
2Daily SDK → TavusWebRTCReal-time audio/video stream sent to Tavus cloud for processing
3Tavus → Daily SDKtool_callTavus Persona detects patient data intent, emits tool_call event via app-message
4Daily SDK → FrontendEventassistant.js receives app-message event with tool_call payload
5Frontend → ngrokHTTPS POSTFrontend sends HTTP request to ngrok public URL with patient data payload
6ngrok → FastAPITunnelngrok tunnels request from public HTTPS to localhost:8000
7FastAPI → PostgreSQLSQLBackend identifies patient, executes INSERT or UPDATE via psycopg2 pool
8PostgreSQL → FastAPIDataDatabase returns query results — patient record or confirmation
9FastAPI → ngrokJSONBackend sends structured JSON response back through ngrok tunnel
10ngrok → FrontendJSONngrok forwards the JSON response back up to Frontend (assistant.js)
11Frontend → TavusEcho MsgFrontend sends response text to Tavus; avatar speaks it with lip-synced video
↩️Tavus → PatientVideo+AudioAI avatar streams lip-synced video and TTS audio back to patient's browser
